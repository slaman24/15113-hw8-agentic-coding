This percentage was a lot higher than I thought it would be, and I was genuinely impressed with the agent's ability to follow the spec with immense attention to detail.

2. Besides accepting its file creation and terminal command prompts to ensure the code was compiling, I did not have to intervene. I let the agent run in its entirety, and then I opened a new chat and ran a new agent to conduct a review on the code. I did spend a good amount of time developing and editing SPEC.md, so I think the detail I was able to include prevented me from having to step in during Phase 2.

3. I thought the AI review was extremely helpful and caught bugs that I do not believe I would have been able to catch myself. For example, the review noticed that while there was error handling for an empty question bank, there was no error handling for an unreadable question bank (permissions, I/O failure). This was not something I originally thought to account for, so I was impressed with the extensiveness and thoroughness of the AI review. Additionally, the AI review noticed that while passwords were being stored securely, scores were not based on the way they were being hashed. As someone not as familiar with information security and privacy, this was extremely valuable to be informed about. I also appreciated that the AI review flagged code quality (import that was not used) as well as visible on-screen password entry. I feel like these examples show how in-depth and detailed the review was, especially when it came to information security.

4. Overall, I feel that my SPEC.md contained a good level of detail that helped the output turn out the way I wanted it to. I also feel in some ways the AI went above and beyond to implement the spec. For example, one thing I tried to emphasize was robustness with user input, as I wanted the quiz app to be as user-friendly and forgiving as possible. For this specifically, I wanted it to be case insensitive so 'A' and 'a' would both count as valid input for a multiple choice question. The AI definitely delivered on this, and it also gave the user the option to type the answer out as well (for example 'List' or 'list'). I was really happy with this attention to detail and consideration for good UI/UX design.

5. Based on my experience, I feel that this plan-delegate-review approach is better than the conversational back-and-forth when you have a complete vision of what you want to create from the start. You can write up a detailed spec, and then essentially have it built entirely by AI in one go (may have to make some slight modifications based on the results of the review). This plan-delegate-review approach may also be better when you have to be particularly mindful about security or have specific acceptance criteria in mind for a project. On the other hand, I feel that the conversational back-and-forth may be better when you have an idea that you want to iterate on or you want to be more involved in the building process/code development.
